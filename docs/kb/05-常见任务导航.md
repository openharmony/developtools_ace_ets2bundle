# 常见任务导航

## 1. 新增/修改装饰器

### 动态工具链
1. 在 `compiler/src/pre_define.ts` 添加装饰器常量
2. 在 `compiler/src/process_component_member.ts`（V1）或 `compiler/src/process_struct_componentV2.ts`（V2）添加变换逻辑
3. 在 `compiler/src/validate_ui_syntax.ts` 添加校验逻辑
4. 规格：[02-装饰器速查](02-装饰器速查.md) → 对应 01-XX

### 静态工具链
1. 在 `arkui-plugins/common/predefines.ts` 添加 `DecoratorNames` 枚举
2. 在 `arkui-plugins/ui-plugins/property-translators/` 新建 translator 文件
3. 在 `arkui-plugins/collectors/ui-collectors/validators/rules/` 新建 `check-xxx.ts` 校验规则
4. 在 `index.ts` 中 `export *` 注册规则
5. 规格：[02-装饰器速查](02-装饰器速查.md) → 对应 01-XX 和 02-XX

## 2. 新增/修改内置组件
1. 在 `compiler/components/` 新建/修改 JSON 文件（遵循 schema）
2. 在 `compiler/src/component_map.ts` 确认自动加载（`require()` 机制）
3. 在 `compiler/src/pre_define.ts` 添加控制流常量（如需要）
4. 阅读 `compiler/如何新增或修改组件指导规范.md`
5. 规格：[04-组件速查](04-组件速查.md) → [01-09 组件转换](../specs/01-编译转换/01-09%20组件转换.md)

## 3. 新增/修改校验规则
1. 在 `arkui-plugins/collectors/ui-collectors/validators/rules/` 新建 `check-xxx.ts`
2. 继承 `BaseValidator<T, Info>`，实现 `parsed()`/`checked()` 钩子
3. 在 `index.ts` 中 `export *` 注册
4. 在 `compiler/src/validate_ui_syntax.ts` 添加动态侧校验（如需要）
5. 规格：[03-校验规则速查](03-校验规则速查.md) → 对应 02-XX

## 4. 修改 struct 转换逻辑
1. 动态（V1）：`compiler/src/process_component_class.ts`（struct → class）
2. 动态（V2）：`compiler/src/process_struct_componentV2.ts`（V2 struct）
3. 静态：`arkui-plugins/ui-plugins/component-transformer.ts`（parsed 阶段）
4. 规格：[01-01 struct转换](../specs/01-编译转换/01-01%20struct转换.md)

## 5. 修改属性转换逻辑
1. 动态：`compiler/src/process_component_build.ts`（`bindComponentAttr`/`addComponentAttr`）
2. 规格：[01-03 通用属性转换](../specs/01-编译转换/01-03%20通用属性转换.md) ~ [01-08 Modifier属性转换](../specs/01-编译转换/01-08%20Modifier属性转换.md)（按属性类型选择）
