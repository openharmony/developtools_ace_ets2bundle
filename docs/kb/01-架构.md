# 双工具链架构

## 架构边界

| 维度 | 动态工具链（compiler/） | 静态工具链（arkui-plugins/） |
|---|---|---|
| AST 基础 | TypeScript Compiler API | es2panda AST（`@koalaui/libarkts`） |
| 转换方式 | 声明式 → 命令式 create/pop 序列 | struct → class + property translator |
| 状态管理 | `ObservedPropertySimple` 等包装类 | `STATE_MGMT_FACTORY.makeXxx` 工厂 |
| 组件定义 | `components/*.json`（145 个）+ `component_map.ts` | `predefines.ts` 枚举 + `arkuiImportList.ts`（165 个） |
| 校验入口 | `validate_ui_syntax.ts`（数字错误码 10905xxx） | `collectors/ui-collectors/validators/rules/`（规则名 + FixSuggestion） |
| 规格文档 | `docs/specs/01-编译转换/`（71 篇） | 同左（每篇含动态+静态双章节） |

**核心不变量：两套工具链运行时不交叉依赖。**

## 插件流水线

### 动态工具链（compile_plugin.js，顺序固定）
```
memoryMonitor → watchChangeFiles → etsChecker → visualTransform → etsTransform → apiTransform → genAbc → terserPlugin → babelPlugin → createProgramPlugin
```

### 静态工具链（顺序固定）
1. interop-plugin（parsed：struct → class）
2. ui-plugin（parsed：ComponentTransformer + checked：Collector → CheckedTransformer）
3. ui-syntax-plugin（checked：lint 规则）
4. memo-plugin（checked：unmemoize + recheckSubtree）

## 编译模式

| 模式 | 适用场景 | 输出 | 关键差异 |
|---|---|---|---|
| ESMODULE | HAR/HSP 包 | 按模块独立 .abc | 无 babelPlugin |
| JSBUNDLE | 应用发布 | 单一 .abc 包 | 激活 babelPlugin CommonJS 变换 |
