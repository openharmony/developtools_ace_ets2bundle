# 功能概述
声明合并（Declaration Merger）将 HAR 包中分散的 `.d.ets`/`.d.ts` 声明文件合并为单一入口声明文件，通过双向映射在源文件路径和声明缓存路径之间解析模块引用，收集导出实体、类型依赖和系统 API 实体，生成主文件和伴生文件（companion file）。

### 关键转换逻辑

#### 1. resolveWithFallback（line 25-39）
```typescript
function resolveWithFallback(
  moduleName: string,
  sourceContainingFile: string | null,
  resolved: ts.ResolvedModuleFull | null
): ts.ResolvedModuleFull | null {
  if (resolved) return resolved;          // 优先使用原始解析结果
  if (!sourceContainingFile) return null;
  // 回退：从源文件路径解析模块
  const fallback = resolveModuleNamesOrig([moduleName], sourceContainingFile);
  return fallback[0] ?? null;
}
```
- 当原始 `resolveModuleNames` 返回 null 时，从源文件路径（非声明缓存路径）重新解析。

#### 2. buildBidirectionalMaps（line 41-54）
```typescript
function buildBidirectionalMaps() {
  const sourceToDecl = new Map();  // 源文件路径 → 声明缓存路径
  const declToSource = new Map();  // 声明缓存路径 → 源文件路径
  harFilesRecord.forEach(value => {
    if (value.originalDeclarationCachePath) {
      sourceToDecl.set(toUnixPath(value.sourcePath), toUnixPath(value.originalDeclarationCachePath));
      declToSource.set(toUnixPath(value.originalDeclarationCachePath), toUnixPath(value.sourcePath));
    }
  });
  return { sourceToDecl, declToSource };
}
```
- 遍历 `harFilesRecord`（HAR 文件记录），构建双向映射。
- `sourceToDecl`：解析源文件到声明缓存时使用。
- `declToSource`：将声明缓存文件解析请求映射回源文件路径。

#### 3. createDeclarationModuleResolver（line 56-89）
```
1. 原始 resolveModuleNames 解析模块
2. 通过 declToSource 将 containingFile 从声明路径映射回源路径
3. 若解析失败，调用 resolveWithFallback 从源路径回退解析
4. 通过 sourceToDecl 将解析结果的 resolvedFileName 替换为声明缓存路径
5. 设置正确的 extension（.d.ets → Dets，.d.ts → Dts）
```

#### 4. DeclarationMerger 类（line 165-239）

#### 5. mergeEntry 流程
```
resetState()
  → 读取入口 SourceFile
  → 设置 entryExt 和 entryFilePath
  → 收集导出符号
  → 收集类型依赖（深度限制 50）
  → 收集系统 API 实体
  → 构建 EmitContext
  → 生成主文件行（primaryLines）
  → 生成伴生文件行（companionLines）
  → 写入主文件和伴生文件
```

#### 6. EntityKind 枚举
| 值 | 说明 |
|---|---|
| `Exported` | 直接导出的符号 |
| `TypeDependency` | 类型依赖（其他类型引用的类型） |
| `SystemApiImport` | 系统 API 导入（如 `@ohos.xxx`） |
| `SystemApiReexport` | 系统 API 再导出 |

#### 7. EmitContext
| 字段 | 说明 |
|---|---|
| `primaryLines` | 主文件行 |
| `companionLines` | 伴生文件行 |
| `emittedStatements` | 已输出的语句文本集合（去重） |
| `crossExtExportedNames` | 跨扩展名导出的名称 |
| `crossExtImportNamespaces` | 跨扩展名导入的命名空间 |
| `primaryExt` | 主文件扩展名 |
| `companionModuleSpecifier` | 伴生文件模块说明符 |
