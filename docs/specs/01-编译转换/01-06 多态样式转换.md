# 功能概述
将 `.stateStyles({...})` 多态样式属性转换为 `ViewStackProcessor.visualState(stateName)` / `ViewStackProcessor.visualState()` 命令式调用，包裹不同状态下的属性。支持 `pressed`、`normal`、`hovered`（API ≥ 26）、`focused`、`selected`、`disabled 等状态。动态工具链对 `stateStyles` 对象的每个属性递归展开，支持 `@Styles` 方法引用、内联对象字面量和全局样式函数引用三种形式。
| 转换规则 | 说明 |
|---|---|
| 入口条件 | `temp.arguments.length === 1 && ts.isObjectLiteralExpression(temp.arguments[0])` |
| visualState 包裹 | 每个状态属性前后生成 `visualState(stateName)` / `visualState()` |
| reverse 展开 | `temp.arguments[0].properties.reverse()` 保证多状态属性的正确执行顺序 |
| hovered 兼容 | `hovered` 状态在 API < 26 时跳过，且映射为 `hover` 名称 |
| @Styles 引用展开 | `this.xxx` 引用 `@Styles` 方法时递归 `bindComponentAttr` |
| 内联对象限制 | 仅展开 `properties.length === 1` 且为 `PropertyAssignment` 的内联对象 |
| 空对象跳过 | `properties.length === 0` 的内联对象跳过不报错 |

## 动态
### 源码参考位置

- `compiler/src/process_component_build.ts:3329-3343`（`createViewStackProcessor`，visualState 生成）
- `compiler/src/process_component_build.ts:3345-3396`（`traverseStateStylesAttr`，状态属性遍历展开）
- `compiler/src/pre_define.ts:212`（`ATTRIBUTE_STATESTYLES = 'stateStyles'`）

### 转换前的原始代码
```typescript
Button('ok')
  .stateStyles({
    pressed: Button('pressed').width(100),
    normal: this.normalStyle,
    hovered: this.hoverStyle
  })
```

### 转换后的代码（Legacy 和 Partial Update 一致）
```typescript
Button.create('ok')
ViewStackProcessor.visualState('normal')         // reverse 后 normal 先展开
// normalStyle 内容（@Styles 方法展开）
ViewStackProcessor.visualState()                 // 结束 normal 状态
ViewStackProcessor.visualState('hover')          // hovered 映射为 hover
// hoverStyle 内容（@Styles 方法展开）
ViewStackProcessor.visualState()                 // 结束 hover 状态
ViewStackProcessor.visualState('pressed')
Button.create('pressed')
Button.width(100)
Button.pop()
ViewStackProcessor.visualState()                 // 结束 pressed 状态
Button.pop()
```

### 关键转换逻辑
- **入口条件**（line 3059-3070）：
  - `propName === ATTRIBUTE_STATESTYLES`（line 3059）：匹配 `stateStyles` 属性。
  - `temp.arguments.length === 1 && ts.isObjectLiteralExpression(temp.arguments[0])`（line 3060）：参数为单个对象字面量。
  - 生成初始 `createViewStackProcessor(temp, true)`（line 3061）：`endViewStack = true` 时不传参数，生成空 `visualState()` 调用。
  - Recycle 组件特殊处理（line 3062-3064）：将初始 visualState 也推入 `updateStatements`。
  - 调用 `traverseStateStylesAttr`（line 3065-3066）遍历展开各状态属性。
  - 参数不为对象字面量时调用 `validateStateStyleSyntax` 报错（line 3069）。

- **`createViewStackProcessor`**（line 3329-3343）：
  - `endViewStack = true`：生成 `ViewStackProcessor.visualState()`（无参数，结束状态）。
  - `endViewStack = false` 且 `item.name` 存在：
    - `item.name.getText() === HOVERED` 时映射为 `HOVER`（`'hover'`）。
    - 其他状态名原样使用。
    - 生成 `ViewStackProcessor.visualState('stateName')`。

- **`traverseStateStylesAttr`**（line 3345-3396）：完整展开逻辑。
  - **`properties.reverse()`**（line 3348）：反转属性顺序。因 `ViewStackProcessor` 是栈结构，反转后 `normal` 先入栈 `pressed` 后入栈，使 `pressed` 在最内层。
  - **hovered API 兼容**（line 3349-3352）：
    ```typescript
    if (item.name && item.name.getText && item.name.getText() === HOVERED &&
      !isCompatibleVersionOverTarget(26)) {
      return;  // 跳过 hovered 属性
    }
    ```
    API < 26 时直接 `return` 跳过 `hovered` 状态。
  - **`@Styles` 方法引用**（line 3353-3371）：两种引用形式：
    - `this.xxx` 属性访问（line 3353-3362）：`ts.isPropertyAccessExpression(item.initializer)` 且 `item.initializer.expression.getText() === THIS` 且 `INNER_STYLE_FUNCTION.get(item.initializer.name.getText())` 存在时，递归调用 `bindComponentAttr` 展开 `@Styles` 方法内容。Recycle 组件时同时展开到 `updateStatements`。
    - 全局样式标识符引用（line 3363-3371）：`ts.isIdentifier(item.initializer)` 且 `GLOBAL_STYLE_FUNCTION.get(item.initializer.getText())` 存在时，同样递归 `bindComponentAttr`。
  - **内联对象字面量展开**（line 3372-3382）：
    - 条件：`ts.isObjectLiteralExpression(item.initializer)` 且 `item.initializer.properties.length === 1` 且 `ts.isPropertyAssignment(item.initializer.properties[0])`。
    - **只展开第 1 条语句限制**：`properties.length === 1` 限制意味着多属性内联对象不展开，直接走到 else 分支报错。
    - 展开方式：取 `item.initializer.properties[0].initializer`，构造 `ExpressionStatement` 后递归 `bindComponentAttr`。
  - **空对象跳过**（line 3383-3386）：
    ```typescript
    if (!(ts.isObjectLiteralExpression(item.initializer) && item.initializer.properties.length === 0)) {
      validateStateStyleSyntax(temp, log);  // 非空非单属性对象报错
    }
    ```
    `properties.length === 0` 的空对象不报错，直接跳过。
  - **状态包裹**（line 3388-3394）：每个属性展开后，调用 `createViewStackProcessor(item, false)` 生成 `visualState('stateName')` 调用，推入 `statements`（和 Recycle 时的 `updateStatements`）。

## 静态
### 源码参考位置
静态工具链中多态样式通过通用 builder lambda 链式调用展开处理，无独立的 stateStyles 转换逻辑。stateStyles 作为 `CommonMethod` 接口的方法调用保留。

### 转换前的原始代码
同动态工具链

### 转换后的代码
stateStyles 属性作为组件方法调用保留在 builder lambda 体中，由运行时框架解释执行。

## 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| visualState 机制 | 显式生成 `ViewStackProcessor.visualState(name)/visualState()` | 无此机制 |
| @Styles 引用 | 递归 `bindComponentAttr` 展开 @Styles 方法内容 | 保留原始引用 |
| hovered 状态 | API < 26 跳过，名称映射为 `hover` | 无版本兼容处理 |
| reverse 顺序 | `properties.reverse()` 保证栈顺序正确 | 无此机制 |
| 内联对象限制 | 仅展开 `properties.length === 1` 的单属性对象 | 无此限制 |
