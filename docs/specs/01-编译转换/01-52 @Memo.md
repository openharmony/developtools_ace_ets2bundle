# 功能概述
`@Memo` 装饰器标记函数/方法进行缓存优化，由 memo-plugins 在 checked 阶段执行 "unmemoize" 变换，将 `@memo` 函数展开为带缓存的命令式调用。

## 动态
### 源码参考位置
动态工具链不直接处理 @Memo 变换。@Memo 是静态工具链特有的优化机制，通过 ui-plugins 在多处为函数添加 `@memo` 注解，让 memo-plugins 后续介入处理。

### 转换前的原始代码
```typescript
@memo
function compute(a: number, b: number): number {
  return a + b
}
```

### 转换后的代码
动态工具链不直接处理 @Memo 变换。@Memo 是静态工具链特有的优化机制，通过 ui-plugins 在多处为函数添加 `@memo` 注解，让 memo-plugins 后续介入处理。

### 关键转换逻辑
- 动态工具链不涉及 @Memo 变换
- @Memo 装饰器仅由静态工具链的 memo-plugins 在 checked 阶段处理
- ui-plugins 在多处调 `addMemoAnnotation(node.function)` 为函数添加 `@memo` 注解（如 struct-translators/factory.ts:886 build 方法、entry-translators/factory.ts:247 entry 方法）

## 静态
### 源码参考位置
- `arkui-plugins/memo-plugins/index.ts:31`（`unmemoizeTransform`，插件入口）
- `arkui-plugins/memo-plugins/index.ts:43-77`（`checkedTransform`，checked 阶段处理）
- `arkui-plugins/memo-plugins/function-transformer.ts:83`（`FunctionTransformer`，核心变换器）
- `arkui-plugins/memo-plugins/function-transformer.ts:171-200`（`enter()`，scope 栈管理）
- `arkui-plugins/memo-plugins/utils.ts`（`hasMemoAnnotation`/`isMemoMethodDefinition`，注解检测）

### 转换前的原始代码
```typescript
@memo
function compute(a: number, b: number): number {
  return a + b
}
```

### 转换后的代码
```typescript
// unmemoize: 展开为带缓存的命令式调用
// 基于 PositionalIdTracker + ParameterTransformer + ReturnTransformer + SignatureTransformer + InternalsTransformer
function compute(a: number, b: number): number {
  // 缓存检查逻辑
  // 参数比较
  // 缓存命中返回 / 缓存未命中计算并存储
  // 返回值处理
}
```

### 关键转换逻辑
- `unmemoizeTransform()`（index.ts:31-41）：返回插件对象，注册 `checked` 阶段 `checkedTransform`
- `checkedTransform()`（index.ts:43-77）：
  1. 获取 program 和 AST
  2. 调 `checkedProgramVisit()` 执行 `FunctionTransformer`
  3. **关键**：`arkts.recheckSubtree(script)` — 必须在 AST 修改后调用，触发重新类型检查
  4. `setArkTSAst(script)` 写回 AST
- `FunctionTransformer`（function-transformer.ts:83-800）：
  - scope 栈 `this.scopes: ScopeInfo[]`，每层记录 `{ name, isMemo, regardAsSameScope? }`
  - `enter()` 进入各类节点时压入 scope，记录 `isMemo` 标志
  - 基于 `PositionalIdTracker`（位置 ID 跟踪）+ `ParameterTransformer`（参数变换）+ `ReturnTransformer`（返回变换）+ `SignatureTransformer`（签名变换）+ `InternalsTransformer`（内部变换）协作完成 unmemoize

### 与 ui-plugins 的协作
- ui-plugins 在多处调 `addMemoAnnotation(node.function)` 为函数添加 `@memo` 注解：
  - `struct-translators/factory.ts:886`：build 方法
  - `struct-translators/factory.ts:2306`：`_invokeImpl` 方法
  - `entry-translators/factory.ts:247`：entry 方法
- `NodeCacheFactory.getCache(NodeCacheNames.MEMO)` 跨插件共享 memo 缓存

### recheckSubtree 的必要性
- AGENTS.md 明确要求："@Memo 变换必须调用 `arkts.recheckSubtree()`；跳过 recheck 会导致类型检查过期状态（stale type-check）"
- index.ts:66 严格执行：`arkts.recheckSubtree(script)` 包裹在性能计数器 `memo-recheck` 中
- `arkui-plugins/memo-plugins/function-transformer.ts:94`（`FunctionTransformer` 构造）

## 接口声明交叉验证

### rememberVariable

| 维度 | 说明 |
|---|---|
| SDK 声明 | `stateManagement/` 目录 |
| 功能 | 记忆化变量 |
| 工具链处理 | 运行时 API，编译器不做变换 |
| 关联 | 与 `@Memo` 插件相关，`memo-plugins/function-transformer.ts` 处理 `@Memo` |

`rememberVariable` 作为运行时 API，与 `@Memo` 装饰器的编译期缓存变换互补：`@Memo` 在编译期展开为带缓存的命令式调用，`rememberVariable` 在运行时提供记忆化变量支持。

## 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| @Memo 支持 | 不支持 | memo-plugins 在 checked 阶段执行 unmemoize |
| @memo 注解 | 不适用 | ui-plugins 自动添加 @memo 注解 |
| recheckSubtree | 不适用 | 必须调用，否则类型检查过期 |
| 缓存机制 | 不适用 | PositionalIdTracker + 参数/返回/签名变换 |
