# 功能概述
将 `.animation({...})` 属性转换为 `Context.animation()` 命令式调用，通过 `lastStatement` 机制将动画属性包裹。
## 动态
### 源码参考位置
- `compiler/src/process_component_build.ts:3011-3027`（`addComponentAttr` 中 animation 分支）
- `compiler/src/process_ui_syntax.ts:1162`（`processAnimateToOrImmediately`，animateTo 函数级变换）
- `compiler/src/pre_define.ts:210`（`ATTRIBUTE_ANIMATION = 'animation'`）
- `compiler/src/pre_define.ts:209`（`GLOBAL_CONTEXT = 'Context'`）

### 转换前的原始代码
```typescript
Text('hello')
  .width(100)
  .animation({ duration: 1000 })
  .height(200)
```

### 转换后的代码（Legacy 和 Partial Update 一致）
```typescript
Text.create('hello')
Text.width(100)
Context.animation(null)              // 临时置空，标记后续为动画属性
Text.height(200)                      // 这个属性被"动画包裹"
Context.animation({ duration: 1000 }) // 实际动画参数
Text.pop()
```

### 关键转换逻辑
- `addComponentAttr`（line 3011-3027）的 animation 分支使用 `lastStatement` 缓存机制：
  - 遇到 `.animation` 时，先用 `Context.animation(null)`（line 3012-3015）输出缓存的上一条语句
  - 设置 `lastStatement.statement = Context.animation(args)`（line 3024-3025）
  - `lastStatement.hasAnimationAttr = true`（line 3027）标记动画存在
- 循环结束后（line 2416-2418）：若 `lastStatement.statement && lastStatement.kind`，推入 statements
- `animateTo`/`animateToImmediately` 是函数级变换（`process_ui_syntax.ts:1162`），不走属性链

### 源码参考位置
- `compiler/src/process_ui_syntax.ts:1162-1167`（`processAnimateToOrImmediately`，animateTo callee 替换）

### 转换前代码

```typescript

  this.width = 200
})

// 隐式属性动画
Text('hello')
  .width(this.width)
```

### 转换后代码

```typescript
// 显式动画：callee 替换为 GlobalContext.animateTo
GlobalContext.animateTo({ duration: 1000, curve: Curve.EaseInOut }, () => {
  this.width = 200
})

// 隐式属性动画：animation 属性走 updateStatements 分支
Text.create('hello')
Text.width(this.width)
Text.animation(null)                    // 先 push animationNullNode（占位）
// ... 其他属性 ...
Text.animation({ duration: 1000 })      // lastStatement 替换为 animation 调用
// hasAnimationAttr = true 使属性走 updateStatements 分支
```

### 关键转换逻辑

#### 1. processAnimateToOrImmediately（process_ui_syntax.ts:1162-1167）

```typescript
export function processAnimateToOrImmediately(node: ts.CallExpression): ts.CallExpression {
  return ts.factory.updateCallExpression(node, ts.factory.createPropertyAccessExpression(
    ts.factory.createIdentifier(GLOBAL_CONTEXT),
    ts.factory.createIdentifier(node.expression.escapedText.toString())),
  node.typeArguments, node.arguments);
}
```

- **仅修改 callee**：将 `animateTo(...)` 替换为 `GlobalContext.animateTo(...)`
- 处理 `animateTo` 和 `animateToImmediately` 两种形式
- 参数保持不变（`node.arguments` 原样传递）
- `GlobalContext` 是运行时全局上下文对象，提供动画驱动入口

#### 2. .animation() 属性的 lastStatement 机制（process_component_build.ts:3011-3027）

在 `addComponentAttr` 中处理 `ATTRIBUTE_ANIMATION` 属性：

```typescript
if (propName === ATTRIBUTE_ANIMATION) {
  const animationNullNode: ts.ExpressionStatement = ts.factory.createExpressionStatement(
    createFunction(ts.factory.createIdentifier(GLOBAL_CONTEXT), node,
      [ts.factory.createNull()]));
  if (!lastStatement.statement) {
    if (!(temp.arguments.length === 1 &&
      temp.arguments[0].kind === ts.SyntaxKind.NullKeyword)) {
      statements.push(animationNullNode);
    }
  } else {
    statements.push(lastStatement.statement, animationNullNode);
  }
  lastStatement.statement = ts.factory.createExpressionStatement(createFunction(
    ts.factory.createIdentifier(GLOBAL_CONTEXT), node, temp.arguments));
  lastStatement.kind = false;
  lastStatement.hasAnimationAttr = true;
}
```

**处理流程**：

1. **创建 animationNullNode**：生成 `GlobalContext.animation(null)` 语句
2. **判断 lastStatement 是否已有语句**：
   - 如果 `lastStatement.statement` 为空（无前置属性语句）：
     - 如果 `.animation()` 参数不是 `{ null }`，push `animationNullNode` 作为占位
   - 如果 `lastStatement.statement` 非空：
     - 先 push `lastStatement.statement`（前置属性）
     - 再 push `animationNullNode`（动画结束标记）
3. **设置 lastStatement**：
   - `lastStatement.statement` = `GlobalContext.animation(args)`（真正的动画属性调用）
   - `lastStatement.kind = false`：标记不需要在后续自动 pop
   - `lastStatement.hasAnimationAttr = true`：标记存在动画属性

#### 3. hasAnimationAttr 对属性重算的影响

`lastStatement.hasAnimationAttr = true` 的关键影响在于：后续属性处理时，属性会被路由到 `updateStatements` 分支而非 `immutableStatements` 分支。

在 `addComponentAttr` 的 else 分支（普通属性处理，line 3092-3111）中：

```typescript
} else {
  // ...
  const attrStatement = ts.factory.createExpressionStatement(
    createFunction(identifierNode, node, temp.arguments, isAttributeModifier));
  statements.push(attrStatement);
  if ((isRecycleComponent || isReuseComponentInV2) && (!isStylesAttr || isStyleFunction) &&
    !isGestureType(identifierNode) && filterRegularAttrNode(temp.arguments)) {
    immutableStatements.push(attrStatement);
  } else {
    updateStatements.push(attrStatement);
  }
}
```

当 `hasAnimationAttr` 为 true 时，动画期间的属性需要能够被重新计算（re-evaluate），因此属性被放入 `updateStatements` 分支，在每次 rerender 时重新执行，而非作为 immutable 语句只执行一次。

### 设计意图

- **animateTo 替换为 GlobalContext.animateTo**：统一动画驱动入口，确保运行时通过全局上下文管理动画状态
- **.animation() 的 null 占位机制**：在属性链中插入 `animation(null)` 作为动画结束标记，确保属性链的正确嵌套
- **hasAnimationAttr 触发 updateStatements**：动画期间的属性值可能变化，需要在 rerender 时重新计算，因此路由到 update 分支

### 静态变换逻辑

静态工具链中 `.animation()` 通过 `builder-lambda-translators/factory.ts` 的 `updateAnimation` 方法拆分为 `animationStart` + `animationStop`。

#### 源码参考位置
- `arkui-plugins/ui-plugins/builder-lambda-translators/factory.ts:1055-1088`（`updateAnimation`）

```typescript
static updateAnimation(instanceCalls: InstanceCallInfo[]): void {
  let lastAniIdx = 0;
  let curIdx = 0;
  while (curIdx < instanceCalls.length) {
    // ...
    const property = instanceCalls[curIdx].call.callee as arkts.Identifier;
    if (property.name === AnimationNames.ANIMATION) {
      const aniStart = arkts.factory.createCallExpression(
        arkts.factory.createIdentifier(AnimationNames.ANIMATION_START), ...);
      const aniStop = arkts.factory.createCallExpression(
        arkts.factory.createIdentifier(AnimationNames.ANIMATION_STOP), ...);
      instanceCalls.splice(lastAniIdx, 0, { isReceiver: false, call: aniStart });
      instanceCalls[curIdx + 1] = { isReceiver: false, call: aniStop };
      curIdx += 2;
      lastAniIdx = curIdx;
    } else {
      curIdx++;
    }
  }
}
```

- 将 `.animation(args)` 拆分为 `.animationStart(args)` + `.animationStop(args.clone())`
- `animationStart` 插入到动画属性之前
- `animationStop` 替换原 animation 调用位置

### 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| animateTo 处理 | callee 替换为 `GlobalContext.animateTo` | 由 builder-lambda 链处理 |
| .animation() 拆分 | 不拆分，使用 null 占位 + hasAnimationAttr | 拆分为 `animationStart` + `animationStop` |
| 属性重算路由 | `hasAnimationAttr` 路由到 updateStatements | 由 builder-lambda 链式展开自动处理 |
| 占位机制 | `GlobalContext.animation(null)` | `animationStart` / `animationStop` 配对 |
