# 功能概述
涵盖事件属性（`.onClick`/`.onChange`/`.onDelete` 等）的转换规则。事件属性不属于 `stateStyles`、`@Extend`、`@AnimatableExtend`、`animation`、`gesture` 等特殊分支，走通用 `else` 分支生成 `Component.attr(args)` 命令式调用。事件属性与普通属性共享同一转换路径，但不做 immutable/update 分离——除非处于 Recycle/Reuse 组件且参数为常规属性时才进入 `immutableStatements`。

| 转换规则 | 说明 |
|---|---|
| 事件属性 | `.onClick`/`.onChange`/`.onDelete` 等走通用 `else` 分支，生成 `Component.attr(args)` |
| 与普通属性的区别 | 无独立转换分支；不强制 immutable/update 分离；非 modifier 时不追加 `bind(this)` |
| Recycle/Reuse 特殊处理 | 满足条件时事件属性也进入 `immutableStatements`，否则进 `updateStatements` |

### 关键转换逻辑
- 通用 `else` 分支（line 3092-3111）：事件属性不被前面的 `animation`、`gesture`、`@Extend`、`stateStyles`、`$$`/`!!` 双向绑定等分支匹配时，落入此分支。
- `loopEtscomponent`（line 3093）：遍历参数数组，将 `EtsComponentExpression` 节点转为 `CallExpression`，处理内联组件表达式参数。
- 过滤 `reuseId`/`reuse` 属性（line 3094）：`propName !== RECYCLE_REUSE_ID && propName !== REUSE_ATTRIBUTE` 时才生成语句。
- `isAttributeModifier` 判定（line 3095-3099）：仅 `attributeModifier`/`contentModifier`/`menuItemContentModifier` 为 `true`，事件属性始终为 `false`。
- `createFunction` 调用（line 3100-3101）：`createFunction(identifierNode, node, temp.arguments, isAttributeModifier)`，事件属性 `isAttributeModifier = false`。
- `createFunction` 内部逻辑（line 3872-3917）：
  - `globalThisArr = [COMPONENT_GESTURE, GLOBAL_CONTEXT]`（line 3876）：Gesture 和 Context 类型组件通过 `globalThis.xxx` 访问，其他组件直接用组件名。
  - 事件属性 `type` 不是 `COMPONENT_CREATE_FUNCTION`，跳过 `processDollarEtsComponent`/`processExclamationEtsComponent`/`transformBuilder`/`navigationCreateParam` 等创建参数特殊处理。
  - `isAttributeModifier = false` 时（line 3909-3913）：生成 `Component.attr(args)`，即 `ts.factory.createPropertyAccessExpression(node, attrNode)` 作为调用目标。
  - `isAttributeModifier = true` 时（line 3898-3908）：生成 `Component.attr.bind(this)(args)`，事件属性不走此路径。
- immutable/update 分离（line 3103-3108）：
  - 当 `isRecycleComponent || isReuseComponentInV2` 且 `!isStylesAttr || isStyleFunction` 且 `!isGestureType(identifierNode)` 且 `filterRegularAttrNode(temp.arguments)` 为 true 时，事件属性进入 `immutableStatements`。
  - 否则进入 `updateStatements`（常规情况）。
  - `isGestureType`（line 3114-3116）：`GESTURE_TYPE_NAMES.has(node.escapedText.toString())`，手势类型组件的事件属性不进入 immutable。
  - `filterRegularAttrNode`（line 3118）：所有参数都不是 EtsComponent 表达式时返回 true。

## 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| 事件属性 | 生成 `Component.attr(args)` 命令式调用 | 保留在 builder lambda 体中 |
| immutable/update 分离 | Recycle/Reuse 组件中可能分离 | 无此机制 |
| bind(this) | 事件属性不追加（仅 modifier 属性追加） | 无此机制 |
