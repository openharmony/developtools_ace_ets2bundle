# 功能概述
将 `!!` 非空双向数据绑定语法（V2，如 `TextInput({text: !!this.message})`）转换为 `{ value: varExp, $value: (newValue) => { varExp = newValue } }` 对象。`!!` 语法在动态工具链中是 V2 双向绑定机制，与 V1 的 `$$` 语法类似但覆盖更多组件/属性，且回调名不同（`$value` vs `changeEvent`）。
| 转换规则 | 说明 |
|---|---|
| 语法检测 | `isDoubleExclamationToChange` 检测 `!!expr` 模式（`ts.isNonNullExpression` 嵌套） |
| 对象生成 | `generateObjectForExclamation` 生成 `{ value: varExp, $value: (newValue) => { varExp = newValue } }` |
| 回调名 | `$value`（`$_VALUE = '$value'`），带 `$` 前缀 |
| 与 `$$` 的区别 | `$$` 使用 `changeEvent`（`$$_CHANGE_EVENT`）作为回调名；`!!` 使用 `$value` |
| 支持组件 | `PROPERTIES_ADD_DOUBLE_EXCLAMATION` Map（组件 → 属性集合） |
| 支持样式 | `STYLE_ADD_DOUBLE_EXCLAMATION` Set（bindMenu/bindContextMenu/bindPopup/bindContentCover/bindSheet） |
| 参数提取 | `updateArgumentForExclamation` 剥离 `!!` 非空表达式，提取内部变量 |

### 关键转换逻辑
- **`isDoubleExclamationToChange`**（line 3210-3217）：检测 `!!expr` 模式。
  ```typescript
  function isDoubleExclamationToChange(isStylesAttr, identifierNode, propName, temp): boolean {
    return !isStylesAttr &&
      (STYLE_ADD_DOUBLE_EXCLAMATION.has(propName) && temp.arguments.length && temp.arguments[0] ||
      PROPERTIES_ADD_DOUBLE_EXCLAMATION.has(identifierNode.escapedText.toString()) &&
      PROPERTIES_ADD_DOUBLE_EXCLAMATION.get(identifierNode.escapedText.toString()).has(propName)) &&
      ts.isNonNullExpression(temp.arguments[0]) && ts.isNonNullExpression(temp.arguments[0].expression) &&
      !ts.isNonNullExpression(temp.arguments[0].expression.expression);
  }
  ```
  四个条件：
  1. `!isStylesAttr`：非 @Styles 上下文。
  2. 属性在 `STYLE_ADD_DOUBLE_EXCLAMATION` 中或组件-属性在 `PROPERTIES_ADD_DOUBLE_EXCLAMATION` 中。
  3. `ts.isNonNullExpression(temp.arguments[0])`：第一个参数是 `!!expr` 非空表达式。
  4. `ts.isNonNullExpression(temp.arguments[0].expression)`：内部也是非空表达式（即 `!!` 双层嵌套）。
  5. `!ts.isNonNullExpression(temp.arguments[0].expression.expression)`：最内层不是非空表达式（防止 `!!!` 三层）。

- **`classifyArgumentsNumV2`**（line 3270-3279）：参数分类和对象生成。
  ```typescript
  function classifyArgumentsNumV2(args, argumentsArr, propName, identifierNode): void {
    const componentName: string = identifierNode.escapedText.toString();
    if (STYLE_ADD_DOUBLE_EXCLAMATION.has(propName) && args.length ||
    PROPERTIES_ADD_DOUBLE_EXCLAMATION.has(componentName) && args.length === 1 &&
    PROPERTIES_ADD_DOUBLE_EXCLAMATION.get(componentName).has(propName)) {
      const varExp: ts.Expression = updateArgumentForExclamation(args[0]);
      argumentsArr.push(generateObjectForExclamation(varExp), ...args.slice(1));
    }
  }
  ```
  - 调用 `updateArgumentForExclamation` 提取变量表达式。
  - 调用 `generateObjectForExclamation` 生成双向绑定对象。
  - 追加到 `argumentsArr`，保留其余参数 `...args.slice(1)`。

- **`updateArgumentForExclamation`**（line 2929-2934）：剥离 `!!` 非空表达式。
  ```typescript
  function updateArgumentForExclamation(argument): ts.Expression {
    if (ts.isNonNullExpression(argument) && ts.isNonNullExpression(argument.expression)) {
      return argument.expression.expression;  // !!this.message → this.message
    }
    return argument;
  }
  ```
  剥离两层 `NonNullExpression`，返回最内层变量表达式。

- **`generateObjectForExclamation`**（line 3281-3304）：生成双向绑定对象。
  ```typescript
  function generateObjectForExclamation(varExp: ts.Expression): ts.ObjectLiteralExpression {
    return ts.factory.createObjectLiteralExpression([
      ts.factory.createPropertyAssignment(
        ts.factory.createIdentifier($$_VALUE),    // 'value'
        varExp),                                   // this.message
      ts.factory.createPropertyAssignment(
        ts.factory.createIdentifier($_VALUE),      // '$value'
        ts.factory.createArrowFunction(
          undefined, undefined,
          [ts.factory.createParameterDeclaration(
            undefined, undefined,
            ts.factory.createIdentifier($$_NEW_VALUE),  // 'newValue'
            undefined, undefined, undefined)],
          undefined,
          ts.factory.createToken(ts.SyntaxKind.EqualsGreaterThanToken),
          ts.factory.createBlock(
            [ts.factory.createExpressionStatement(ts.factory.createBinaryExpression(
              varExp,                                    // this.message
              ts.factory.createToken(ts.SyntaxKind.EqualsToken),
              ts.factory.createIdentifier($$_NEW_VALUE)  // newValue
            ))], false)
        ))
    ], false);
  }
  ```
  生成两个属性：
  1. `value: varExp` — 当前值引用（如 `this.message`）。
  2. `$value: (newValue) => { varExp = newValue }` — 赋值回调（如 `(newValue) => { this.message = newValue }`）。

- **与 `$$` 语法的关键差异**：
  - **回调名不同**：`!!` 使用 `$value`（`$_VALUE = '$value'`，line 388）；`$$` 使用 `changeEvent`（`$$_CHANGE_EVENT = 'changeEvent'`，line 387）。
  - **值名相同**：两者都使用 `value`（`$$_VALUE = 'value'`，line 386）。
  - **参数分类函数不同**：`!!` 使用 `classifyArgumentsNumV2`（line 3270）；`$$` 使用 `classifyArgumentsNum`（line 3255 附近）。
  - **对象生成函数不同**：`!!` 使用 `generateObjectForExclamation`（line 3281）；`$$` 使用 `generateObjectForDollar`（line 3306）。
  - **覆盖范围不同**：`PROPERTIES_ADD_DOUBLE_EXCLAMATION` 比 `PROPERTIES_ADD_DOUBLE_DOLLAR` 多出 `Navigation.navBarWidth`、`SideBarContainer.sideBarWidth`、`Grid.enableEditMode`、`List.enableEditMode`、`Scroll.zoomScale`、`ContainerReader.size/widthBreakpoint/heightBreakpoint`。
  - **样式属性相同**：`STYLE_ADD_DOUBLE_EXCLAMATION` 和 `STYLE_ADD_DOUBLE_DOLLAR` 都包含 `bindPopup`、`bindContentCover`、`bindSheet`，`!!` 额外包含 `bindMenu`、`bindContextMenu`。

- **`PROPERTIES_ADD_DOUBLE_EXCLAMATION`**（pre_define.ts:491-524）：支持 `!!` 的组件/属性映射（共 25 个组件）：
  - `TextInput` → `text`
  - `Navigation` → `navBarWidth`
  - `Scroll` → `zoomScale`
  - `ContainerReader` → `size`/`widthBreakpoint`/`heightBreakpoint`
  - `ListItem` → `selected`
  - `Refresh` → `refreshing`
  - `Slider` → `value`
  - 其他：`Radio.checked`、`TextTimer.format/count/isCountDown`、`Checkbox.select`、`CheckboxGroup.selectAll`、`DatePicker.selected`、`TimePicker.selected`、`MenuItem.selected`、`Panel.mode`、`Rating.rating`、`Search.value`、`SideBarContainer.showSideBar/sideBarWidth`、`Stepper.index`、`Swiper.index`、`Tabs.index`、`TextArea.text`、`TextPicker.value/selected`、`Toggle.isOn`、`AlphabetIndexer.selected`、`ArcAlphabetIndexer.selected`、`Select.selected/value`、`Calendar.date`、`Grid.enableEditMode`、`List.enableEditMode`、`GridItem.selected`

## 静态
### 源码参考位置
静态工具链中 `!!` 作为非空断言操作符，通过 `arkts.factory.createTSNonNullExpression()` 处理（参见 `arkui-plugins/ui-plugins/struct-translators/factory.ts:155`，`createNonNullOrOptionalMemberExpression`）。

在 @Link/@ObjectLink/@Param 等 `isRequired` 或 `canDefinitelyBeNonNull` 场景下使用（见 `arkui-plugins/ui-plugins/property-translators/base.ts:132-135`、`138-140`）。

### 转换后的代码
`!!` 在静态工具链中作为非空断言保留，不生成额外的双向绑定对象。双向绑定通过 `$$` 语法和 `makeBindable` 机制实现。

## 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| 语法含义 | 双向数据绑定（V2） | 非空断言操作符 |
| 输出形式 | `{ value: var, $value: (v) => { var = v } }` | 保留为 `TSNonNullExpression` |
| 支持组件 | `PROPERTIES_ADD_DOUBLE_EXCLAMATION` 映射（25 个组件） | @Link/@ObjectLink/@Param 等非空场景 |
| 与 `$$` 区别 | `$value` 回调名（带 $） vs `changeEvent` | 无直接对应 |
| 覆盖范围 | 比 `$$` 多 Navigation/SideBar/Scroll/Grid/List/ContainerReader | 无 |
