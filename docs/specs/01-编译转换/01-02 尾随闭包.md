# 功能概述
将组件调用的尾随闭包语法（`Component() { ... }`）转换为命令式 create/pop 模式，或将尾随 lambda 体注入最后一个 `@BuilderParam` 属性。

## 动态
### 源码参考位置
- `compiler/src/process_component_build.ts:2232`（`createComponent`）
- `compiler/src/process_component_build.ts:3824`（`createTrailingFunction`，ContainerReader 尾随闭包）
- `compiler/src/process_component_build.ts:855`（尾随闭包校验：必须有一个且仅有一个 @BuilderParam）
- `compiler/src/process_component_build.ts:3872`（`createFunction`，普通属性函数生成）

### 转换前的原始代码
```typescript
Column() {
  Text('hello')
  Button('click')
}

// ContainerReader 尾随闭包
ContainerReader(arr) {
  Text('item')
}
```

### 转换后的代码（Legacy）
```typescript
// Column 容器组件
Column.create()
Text.create('hello')
Text.pop()
Button.createWithLabel('click')
Button.pop()
Column.pop()

// ContainerReader 尾随闭包
ContainerReader.create(arr, () => {
  Text.create('item')
  Text.pop()
})
```

### 转换后的代码（Partial Update）
```typescript
// Column 容器组件
this.observeComponentCreation2((elmtId, isInitialRender) => {
  Column.create()
  Text.create('hello')
  if (!isInitialRender) { Text.pop() }
  Button.createWithLabel('click')
  if (!isInitialRender) { Button.pop() }
  if (!isInitialRender) { Column.pop() }
}, Column)
```

### 关键转换逻辑
- `createComponent`（process_component_build.ts:2232）识别组件名，当为 `ContainerReader`（`pre_define.ts:743`）时调用 `createTrailingFunction`
- `createTrailingFunction`（line 3824）将尾随闭包体包装为 ArrowFunction，生成 `ContainerReader.create(args, () => { ...trailingBody... })`
- 校验（line 855）：使用尾随闭包的组件必须有一个且仅有一个 `@BuilderParam` 属性，否则报错

## 静态
### 源码参考位置
- `arkui-plugins/ui-plugins/builder-lambda-translators/factory.ts:975`（`updateContentBodyInBuilderLambda`）
- `arkui-plugins/ui-plugins/builder-lambda-translators/factory.ts:1093`（`transformBuilderLambda`，`leaf.isTrailingCall` 判定）
- `arkui-plugins/ui-plugins/builder-lambda-translators/factory.ts:1163`（`setBuilderLambdaRange`，尾随闭包位置范围）
- `arkui-plugins/ui-plugins/property-translators/builderParam.ts:40`（`initializeStructWithBuilderLambdaProperty`，尾随闭包注入最后一个 @BuilderParam）

### 转换前的原始代码
```typescript
@Component
struct Parent {
  @BuilderParam content: () => void
  build() { this.content?.() }
}
// 调用：Parent({ count: 1 }) { Text("hi") }   // 尾随闭包
```

### 转换后的代码
```typescript
// 最后一个 @BuilderParam 被尾随闭包体注入
__backing_content = initializers?.content ?? initializers?.content ?? (undefined as any)
// builder lambda 链式调用展开为 instance calls 数组
```

### 关键转换逻辑
- `transformBuilderLambda`（factory.ts:1093）检测 `leaf.isTrailingCall`，判定是否为尾随闭包调用
- 若是尾随闭包，`setBuilderLambdaRange`（line 1163）将新节点的 startPosition/endPosition 取 `node.parent.parent` 的范围（覆盖整个尾随 lambda 表达式）
- `initializeStructWithBuilderLambdaProperty`（builderParam.ts:40）检测 `isLastBuilderParam`，是则生成 `__backing_X = initializers?.content ?? initializers?.X ?? initialValue`（把尾随闭包 `content` 注入最后一个 BuilderParam）

## 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| 尾随闭包体处理 | 提取为独立语句序列，注入 create/pop 之间 | 包装为 lambda，注入 `__backing_<BuilderParam>` |
| ContainerReader 特殊处理 | 生成 `create(args, () => {...})` 尾随闭包参数 | 无 ContainerReader 特殊处理 |
| @BuilderParam 校验 | 校验必须有且仅有一个 @BuilderParam | `isLastBuilderParam` 机制注入最后一个 BuilderParam |
| 位置范围 | 不涉及 | `setBuilderLambdaRange` 覆盖整个尾随 lambda 表达式 |
