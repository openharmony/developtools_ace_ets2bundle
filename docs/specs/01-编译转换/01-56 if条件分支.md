# 功能概述
`if`/`else if`/`else` 条件分支转换为 `If.create()` / `If.branchId(id)` / `If.pop()`（Legacy）或 `ifElseBranchUpdateFunction(id, () => {})`（Partial Update）命令式结构。

### 关键转换逻辑
- `processInnerIfStatement`（line 2084-2105）：校验条件表达式非空（line 2087-2097，错误码 10905208）
- `addIfBranchId`（line 2185-2193）：
  - Legacy：`[If.branchId(id), ...statements]`（line 2190）
  - Partial Update：`[ifElseBranchUpdateFunction(id, () => { ...statements })]`（line 2188）
- else 分支缺失时（line 2151-2174）：Partial Update 模式自动生成空分支 `ifElseBranchUpdateFunction(++id, () => {})`
- else if 链处理（line 2123-2130）：嵌套 if 被包装为 `[If.create(), ifElseBranchFunc(id, [thenStatement]), If.pop()]`（Partial Update）

## 静态
### 源码参考位置

### 转换前的原始代码
同动态工具链

### 转换后的代码
if 语句通过 builder lambda 的尾随闭包机制处理，语句被包入 lambda，保留条件渲染语义。if 分支作用域信息通过 collector 收集后缓存。

### 关键转换逻辑
- `checkIsWithInIfConditionScope`（line 883-891）：判断语句是否在 if 分支作用域内
- if 语句的转换通过 builder lambda 链式调用展开完成，不生成 `If.create()`/`If.pop()` 命令式调用

## 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| if 结构 | `If.create()` / `If.branchId(id)` / `If.pop()` | builder lambda 尾随闭包机制 |
| Partial Update 模式 | `ifElseBranchUpdateFunction(id, () => {})` | 无此函数 |
| 分支 ID | 从 0 递增，else 用 id+1 | 无显式分支 ID |
| else 缺失 | Partial Update 自动生成空分支 lambda | 无此处理 |
