# 功能概述
将 `.gesture()`、`.parallelGesture()`、`.priorityGesture()` 手势属性转换为 `Gesture.create(GesturePriority.XXX)` / `Gesture.pop()` 命令式调用。
## 动态
### 源码参考位置
- `compiler/src/process_component_build.ts:3028`（`addComponentAttr` 中 `GESTURE_ATTRS.has(propName)` 判断入口）
- `compiler/src/process_component_build.ts:3419`（`gestureMap`，手势优先级映射）
- `compiler/src/process_component_build.ts:3425`（`parseGesture`，手势解析）
- `compiler/src/pre_define.ts:294-300`（手势常量）

### 转换前的原始代码
```typescript
Text('hello')
  .gesture(TapGesture().onAction(() => { console.log('tap') }))
  .parallelGesture(SwipeGesture())
  .priorityGesture(LongPressGesture())
```

### 转换后的代码（Legacy 和 Partial Update 一致）
```typescript
Text.create('hello')
Gesture.pop()                              // 清除上一个手势组
Gesture.create(GesturePriority.Low)         // .gesture
TapGesture.create()
TapGesture.onAction(() => { console.log('tap') })
TapGesture.pop()
Gesture.pop()
Gesture.create(GesturePriority.Parallel)   // .parallelGesture
SwipeGesture.create()
SwipeGesture.pop()
Gesture.pop()
Gesture.create(GesturePriority.High)        // .priorityGesture
LongPressGesture.create()
LongPressGesture.pop()
Gesture.pop()
Text.pop()
```

### 关键转换逻辑
- `gestureMap`（line 3419）映射：
  - `priorityGesture` -> `GesturePriority.High`
  - `parallelGesture` -> `GesturePriority.Parallel`
  - `gesture` -> `GesturePriority.Low`
- `parseGesture`（line 3425）：
  - 先 `Gesture.pop()` 清除上一个手势组（line 3427-3429）
  - `parseGestureInterface` 解析手势接口（line 3432）
  - 生成 `Gesture.create(GesturePriority.XXX, [可选第二参数])`（line 3444-3448）
- 常量：`GESTURE_ATTRIBUTE='gesture'`、`PARALLEL_GESTURE_ATTRIBUTE='parallelGesture'`、`PRIORITY_GESTURE_ATTRIBUTE='priorityGesture'`（pre_define.ts:294-296），`GESTURE_ENUM_KEY='GesturePriority'`（pre_define.ts:297）

## 静态
### 源码参考位置
静态工具链中手势属性通过通用 builder lambda 链式调用展开处理，无独立的特殊手势转换逻辑。手势属性作为 `CommonMethod` 接口的方法调用，由 `transformBuilderLambda`（`arkui-plugins/ui-plugins/builder-lambda-translators/factory.ts:1093`）统一展开。

### 转换前的原始代码
同动态工具链

### 转换后的代码
手势属性作为组件方法调用保留在 builder lambda 体中，由运行时框架解释执行。
- 手势接口递归解析：`parseGestureInterface`（line 3432）递归处理嵌套手势（如 TapGesture().onAction()）
- 手势属性检测入口：`process_component_build.ts:3028`（`GESTURE_ATTRS.has(propName)`）
- 手势优先级映射：`gesture` -> `GesturePriority.Low`，`parallelGesture` -> `GesturePriority.Parallel`，`priorityGesture` -> `GesturePriority.High`
- 手势嵌套：子组件的手势先于父组件触发，`parallelGesture` 可与父组件同时触发
- 常量定义：`compiler/src/pre_define.ts:294-300`（`GESTURE_ATTRIBUTE`/`PARALLEL_GESTURE_ATTRIBUTE`/`PRIORITY_GESTURE_ATTRIBUTE`/`GESTURE_ENUM_KEY`）

- 手势属性检测入口：`process_component_build.ts:3028`（`GESTURE_ATTRS.has(propName)` 判断）
- `parseGestureInterface`（line 3432）递归处理嵌套手势
- 手势常量定义：`GESTURE_ATTRIBUTE`/`PARALLEL_GESTURE_ATTRIBUTE`/`PRIORITY_GESTURE_ATTRIBUTE`（`pre_define.ts:294-296`）

- `parseGesture` 先调用 `Gesture.pop()` 清除上一个手势组（`process_component_build.ts:3427`）

## 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| 手势优先级映射 | `gestureMap` 显式映射到 `GesturePriority.Low/Parallel/High` | 无显式映射，由运行时处理 |
| Gesture.pop/create | 显式生成 `Gesture.pop()` 和 `Gesture.create(priority)` | 不生成命令式调用 |
| 手势接口解析 | `parseGestureInterface` 递归解析嵌套手势 | 保留原始声明式调用 |
