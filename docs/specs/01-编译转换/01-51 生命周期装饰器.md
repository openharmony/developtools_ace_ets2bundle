# 功能概述
生命周期装饰器（`@ComponentInit`、`@ComponentAppear`、`@ComponentBuilt`、`@ComponentDisappear`、`@ComponentReuse`、`@ComponentRecycle`、`@ComponentActive`、`@ComponentInactive`）标记 V2 组件的生命周期回调方法，在组件的不同阶段由运行时框架自动调用。

## 动态
### 源码参考位置
- `compiler/src/validate_ui_syntax.ts:687-727`（`checkComponentReuseSyntax`）
- `compiler/src/validate_ui_syntax.ts:3514-3536`（`handleLifecycleDecorator`）

### 转换前的原始代码
```typescript
@ComponentV2
struct MyComponent {
  @ComponentInit onInit() { console.log('init') }
  @ComponentAppear onAppear() { console.log('appear') }
  @ComponentBuilt onDidBuild() { console.log('built') }
  @ComponentDisappear onDisappear() { console.log('disappear') }
  @ComponentReuse aboutToReuse(params: object) { console.log('reuse') }
  @ComponentRecycle aboutToRecycle() { console.log('recycle') }
  @ComponentActive onActive() { console.log('active') }
  @ComponentInactive onInactive() { console.log('inactive') }
  build() { Text('hello') }
}
```

### 转换后的代码（Legacy）
V2 装饰器仅支持 Partial Update 模式，无 Legacy 输出。

### 关键转换逻辑
- `handleLifecycleDecorator`（line 3514-3536）：保留为普通方法，移除装饰器。生命周期逻辑由运行时框架通过装饰器元数据处理。
- `checkComponentReuseSyntax`（line 687-727）：校验 `@ComponentReuse`/`@ComponentRecycle` 仅用于 `@Reusable`/`@ReusableV2` 组件。

### 转换后的代码
```typescript
class MyComponent extends CustomComponentV2 {
  onInit() { console.log('init') }
  onAppear() { console.log('appear') }
  onDidBuild() { console.log('built') }
  onDisappear() { console.log('disappear') }
  aboutToReuse(params: object) { console.log('reuse') }
  aboutToRecycle() { console.log('recycle') }
  onActive() { console.log('active') }
  onInactive() { console.log('inactive') }

  __initializeStruct(initializers, content): void {
    // 注入生命周期观察者调用
    this.__initLifecycleObserver(LifecycleMethodType.ABOUT_TO_APPEAR, "onInit")
    this.__initLifecycleObserver(LifecycleMethodType.ABOUT_TO_APPEAR, "onAppear")
    this.__initLifecycleObserver(LifecycleMethodType.ON_DID_BUILD, "onDidBuild")
    // ...
  }
}
```

### 深度逻辑
- 装饰器到生命周期类型的映射（`LIFECYCLE_DECORATOR_TYPE`，line 38-45）：

  | 装饰器 | 生命周期类型 |
  |---|---|
  | `@ComponentInit` | `ABOUT_TO_APPEAR` |
  | `@ComponentAppear` | `ABOUT_TO_APPEAR` |
  | `@ComponentBuilt` | `ON_DID_BUILD` |
  | `@ComponentDisappear` | `ABOUT_TO_DISAPPEAR` |
  | `@ComponentReuse` | `ABOUT_TO_REUSE` |
  | `@ComponentRecycle` | `ABOUT_TO_RECYCLE` |

- `@ComponentActive`/`@ComponentInactive` 通过 `ActiveInactiveCache` 单独收集（line 57-63）。
- `collectLifecycleMethods`（line 47-70）：收集所有生命周期方法到 `ComponentLifecycleCache`，供 `createInitializeStruct` 注入观察者调用。
- `@ComponentReuse` 方法有参数时记录 `hasReuseParam = true` 和 `reuseParams`。
- 声明文件：`@ComponentInit`/`@ComponentAppear` @since 24；`@ComponentBuilt`/`@ComponentReuse`/`@ComponentRecycle` @since 24；`@ComponentActive`/`@ComponentInactive` @since 26。

## 动静态差异说明

| 维度 | 动态工具链 | 静态工具链 |
|---|---|---|
| 方法处理 | 保留为普通方法，移除装饰器 | 保留方法 + `__initializeStruct` 注入观察者 |
| 生命周期注册 | 运行时框架通过装饰器元数据处理 | `ComponentLifecycleCache` 收集 + `createInitializeStruct` 注入 |
| Active/Inactive | 运行时框架处理 | `ActiveInactiveCache` 单独收集 |
| 声明接口 | 无 | `@ComponentInit`/`@ComponentAppear` @since 24 等 |
