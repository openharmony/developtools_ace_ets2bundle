# 规则
校验组件嵌套关系正确性、复用组件在 V2 中的兼容性、@ComponentV2 中不可使用 @Link 组件。

- 静态：

  - `arkui-plugins/collectors/ui-collectors/validators/rules/check-nested-reuse-component.ts:23`

## 适用对象
- 组件嵌套关系
- @ComponentV2 中的子组件使用
- @Reusable / @ReusableV2 复用组件嵌套

## 报错信息
- 静态：
  - `The component '{{componentName}}' can only have the child component '{{specificChildArray}}'.`
  - `When a custom component is decorated with @ComponentV2 and contains a child component decorated with @Reusable, the child component will not create.`
  - `A V2 component cannot be used with any member property decorated by '@Link' in a V1 component.`
  - `Methods decorated with '@ComponentReuse' in '@ComponentV2' cannot have input parameters.`
  - `A custom component decorated with @Component cannot contain child components decorated with @ReusableV2.`
  - `A custom component decorated with @Reusable cannot contain child components decorated with @ReusableV2.`
  - `A custom component decorated with @ReusableV2 cannot contain child components decorated with @Reusable.`
  - `The template attribute of the Repeat component cannot contain any custom component decorated with @ReusableV2.`

## 错误码
- 无数字错误码（静态工具链使用规则名标识）

## 核心校验规则
1. 组件嵌套关系必须符合 `SPECIFIC_CHILD_COMPONENT`/`SPECIFIC_PARENT_COMPONENT` 映射
2. @ComponentV2 struct 中不可使用 @Reusable(V1) 装饰的子组件
3. @ComponentV2 struct 不可实例化包含 @Link 属性的 V1 组件
4. 复用组件嵌套关系校验
5. reuse 属性校验
6. 校验等级为 ERROR

## 校验实现细节
### 静态工具链实现一：复用组件嵌套（check-nested-reuse-component.ts）
源码位于 `check-nested-reuse-component.ts`（148 行），校验 @Reusable / @ReusableV2 装饰器的嵌套兼容性：

#### 1. 入口与前置条件
`_checkNestedReuseComponent` 检查 metadata 是否同时包含 `structDeclInfo`（子组件声明信息）、`fromStructInfo`（父组件信息）和 `ptr`（CallExpression 节点指针）。三者缺一即返回。

#### 2. Repeat.template 上下文检测（checkHasRepeatOrTemplate）
检测当前 CallExpression 是否处于 `Repeat().template()` 调用链内：
- `findAncestorCallExpression`：沿 `parent` 链向上遍历，跳过中间的 CallExpression（通过 `findNodeInInnerChild` 判断当前节点是否为祖先的内部子节点），找到最外层的 CallExpression
- 验证祖先调用的 callee 是否为 `MemberExpression`：
  - `isTemplateCall`：property 为 `Identifier` 且 name === `'template'`
  - `isRepeatCall`：object 为 CallExpression，其 callee 为 `Identifier` 且 name === `'Repeat'`

#### 3. 四条嵌套规则（validateNestedReuseRules）
从 `fromStructInfo.annotationInfo` 获取父组件装饰器状态，从 `structDeclInfo.annotationInfo` 获取子组件装饰器状态：
- **规则1**：父组件 `hasComponent && !hasReusable`，子组件 `hasReusableV2` → 报错"@Component 不可含 @ReusableV2 子组件"
- **规则2**：父组件 `hasReusable`，子组件 `hasReusableV2` → 报错"@Reusable 不可含 @ReusableV2 子组件"
- **规则3**：父组件 `hasReusableV2`，子组件 `hasReusable` → 报错"@ReusableV2 不可含 @Reusable 子组件"
- **规则4**：处于 Repeat.template 内且子组件 `hasReusableV2` → 报错"Repeat.template 不可含 @ReusableV2 组件"

#### 4. 自动修复建议
每条规则的报错均附带移除组件的修复建议：`createSuggestion('', ...getPositionRangeFromNode(callExpression), 'Remove the component')`，覆盖整个 CallExpression 范围。

### 静态工具链实现二：内置组件嵌套关系（check-nested-relationship.ts）
源码位于 `check-nested-relationship.ts`（298 行），校验内置组件的父子嵌套约束：

#### 1. 入口与组件注册表
`_checkNestedRelationship` 检查 `annotationInfo?.hasComponentBuilder`（内置组件）且无 `structDeclInfo`（非自定义组件），从 `MetaDataCollector.getInstance().componentsInfo` 获取组件注册表。

#### 2. 四类嵌套校验
- **父组件限定**（`checkValidParentComponent`）：沿 parent 链向上查找第一个内置组件 CallExpression，若不在 `validParentComponent` 允许列表中则报错。若中间经过 ForEach/LazyForEach/Repeat 渲染控制组件，降级为 WARN；否则为 ERROR。
- **子组件限定**（`checkValidChildComponent`）：遍历父组件的 ArrowFunction body 中的子组件调用，若不在 `validChildComponent` 列表中则报 ERROR。
- **单子组件**（`checkSingleChildComponent`）：`singleChildComponents` 中的组件只允许一个子节点，超出报 ERROR。
- **原子组件**（`checkNoChildComponent`）：`atomicComponents` 中的组件不允许有任何子节点，否则报 ERROR。

## 适用场景
- V1（@Reusable）和 V2（@ReusableV2）复用机制实现不同，混用会导致组件无法正确创建
- @ComponentV2 使用 V2 状态管理，包含 @Link 的 V1 组件状态管理不兼容
- Repeat.template 有独立的复用缓存机制，与 @ReusableV2 的缓存策略冲突
- 内置组件有严格的父子关系（如 ListItem 必须在 List 内）

## 自动修复建议
- 复用组件嵌套错误：移除不兼容的子组件调用
- @ComponentV2 + @Reusable 子组件：将子组件改为 @ReusableV2 或移除 @Reusable 装饰器
- @ComponentV2 + @Link V1 组件：将 V1 组件改为 V2 装饰器或移除 @Link 属性
- 内置组件嵌套错误：将子组件移至正确的父组件内

## 示例代码
### 反例
```typescript
@ComponentV2
struct V2Comp {
  build() {
    V1ReusableComp()           // V2 中使用 @Reusable V1 组件
  }
}

@Reusable
@Component
struct V1ReusableComp {
  build() { Text('reusable') }
}
```

### 正例
```typescript
@ComponentV2
@ReusableV2
struct V2ReusableComp {
  build() { Text('reusable') }
}

@ComponentV2
struct V2Comp {
  build() {
    V2ReusableComp()          // V2 中使用 @ReusableV2 组件
  }
}
```

## 跨工具链一致性
- 动态工具链（compiler）校验基础组件嵌套关系
- 静态工具链（arkui-plugins）扩展覆盖 V1/V2 复用装饰器嵌套、Repeat.template 限制
- @ReusableV2 相关规则仅静态工具链实现，因 @ReusableV2 是 V2 装饰器
