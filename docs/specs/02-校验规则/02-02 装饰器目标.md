# 规则
校验装饰器只能用于其允许的目标（struct/属性/方法/函数/getAccessor），以及 V1/V2 装饰器不可混用。

## 错误码
- 10905115：方法装饰器用于非方法
- 10905116：@Computed 用于非 getter
- 10905117：函数装饰器冲突
- 10905121：多个内置装饰器
- 10905119：重复装饰器（V2）
- 10905337：装饰器仅用于 struct
- 10905338：V2 装饰器在 V1 struct 中
- 10905339：V1 装饰器在 V2 struct 中
- 10905346：V2 成员装饰器用于非属性

## 核心校验规则
1. `@Computed` 仅用于 GetAccessor
2. `@Monitor`/`@LocalBuilder`/`@SyncMonitor` 仅用于方法
3. V2 成员装饰器（@Local/@Param/@Once/@Event/@Provider/@Consumer）仅用于属性
4. V1 装饰器（@State/@Prop/@Link 等）不可在 @ComponentV2 struct 中使用
5. V2 装饰器不可在 @Component struct 中使用
6. 同一成员不可有多个内置装饰器（排除 @Require/@Once）
7. 函数声明只能有一个内置装饰器（@Builder/@Extend/@AnimatableExtend/@Styles/@Concurrent/@Sendable）

## 示例代码
### 反例
```typescript
@ComponentV2
struct MyComp {
  @State count: number = 0          // V1 装饰器在 V2 struct 中

  @Monitor('count')                 // @Monitor 用于属性而非方法
  prop: number = 0

  @Computed                         // @Computed 用于方法而非 getter
  myMethod(): number { return 1 }
}

@Component
struct V1Comp {
  @Local data: number = 0           // V2 装饰器在 V1 struct 中
}
```

### 正例
```typescript
@ComponentV2
struct MyComp {
  @Local count: number = 0
  @Monitor('count') onCountChange(monitor: IMonitor) { }
  @Computed get doubled(): number { return this.count * 2 }
}

@Component
struct V1Comp {
  @State data: number = 0
}
```

### 静态工具链处理
静态工具链通过 `check-validate-decorator-target.ts` 校验装饰器目标（structOnlyDecorators 不可用于 class/interface/function；propertyOnlyDecorators 不可用于方法），通过 `check-one-decorator-on-function-method.ts` 校验函数上只能使用 @Builder 装饰器。报错通过 `this.report()` 输出，支持 `FixSuggestion`（建议移除注解）。
