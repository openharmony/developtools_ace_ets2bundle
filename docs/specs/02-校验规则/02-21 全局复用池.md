# 规则
校验全局复用池 `poolAccepts` 配置：不可自引用、`poolAccepts` 中的名称必须是 `@Reusable`/`@ReusableV2` 装饰的组件。

## 源码参考位置
- 动态：`compiler/src/validate_ui_syntax.ts`（动态侧无独立规则，poolAccepts 由静态侧校验） 中的全局复用池校验逻辑
- 静态：`arkui-plugins/collectors/ui-collectors/validators/rules/check-global-reuse.ts:32`

## 适用对象
- `@Component`/`@ComponentV2` struct 声明（使用 `poolAccepts` 参数的组件装饰器）

## 报错信息
- 动态：
  - 无对应动态工具链校验
- 静态：
  - `'${structName}' cannot list itself in poolAccepts. The pool is not yet ready when '${structName}' is being constructed`
  - `'${name}' is not a '@Reusable' or '@ReusableV2' component and cannot be added to 'poolAccepts'.`

## 错误码
- 无数字错误码（静态工具链使用规则名标识）

## 核心校验规则
1. `poolAccepts` 数组中不可包含当前 struct 自身的名称（自引用）
2. `poolAccepts` 数组中的每个名称必须解析为一个被 `@Reusable` 或 `@ReusableV2` 装饰的 struct
3. 名称解析范围：同文件 struct 声明 + 导入的 struct 声明

## 示例代码
### 反例
```typescript
@Component({ poolAccepts: ['MyComp'] })
@Reusable
struct MyComp {                     // poolAccepts 中自引用
  build() { Text('hello') }
}

@Component({ poolAccepts: ['NonReusable'] })
@Reusable
struct MyComp2 {
  build() {
    NonReusable()                   // NonReusable 未被 @Reusable/@ReusableV2 装饰
  }
}

struct NonReusable {
  build() { Text('hello') }
}
```

### 正例
```typescript
@Component({ poolAccepts: ['ReusableA', 'ReusableB'] })
@Reusable
struct MyComp {
  build() {
    ReusableA()
    ReusableB()
  }
}

@Reusable
struct ReusableA { build() { Text('a') } }

@ReusableV2
struct ReusableB { build() { Text('b') } }
```

## 校验实现细节

### poolAccepts 自引用检测逻辑
`checkPoolAcceptsNotSelf` 遍历 `poolAccepts` 数组的每个元素，若元素为 `StringLiteral` 且 `elem.str === structName`（当前 struct 名称），报 ERROR：
```
'${structName}' cannot list itself in poolAccepts. The pool is not yet ready when '${structName}' is being constructed
```
原因是 struct 自身构造时复用池尚未就绪，不可将自身加入池中。

### 跨文件 @Reusable/@ReusableV2 解析
`checkPoolAcceptsResolvesToStruct` 对 `poolAccepts` 中每个非自身名称进行解析：
1. 通过 `getETSModule` 获取当前文件的 `ETSModule` AST
2. `collectSameFileStructNames` 收集同文件所有 `ClassDeclaration` 的名称
3. `resolveStructDecl` 解析目标名称：
   - 若名称在同文件 struct 集合中，查找同文件 `ClassDeclaration`
   - 否则通过 `resolveImportedStructDecl` 在 `ImportDeclaration` 的 `ImportSpecifier` 中查找，通过 `getPeerIdentifierDecl` 解析导入的 `ClassDefinition`
4. 解析到的 `ClassDefinition` 必须含 `@Reusable` 或 `@ReusableV2` 装饰器（`hasReusableAnnotation`），否则报错
