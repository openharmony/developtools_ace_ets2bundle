# 规则
校验 `@Once` 装饰器只能装饰成员属性，且必须与 `@Param` 同时使用。

## 补充说明
- `arkui-plugins/common/predefines.ts:235`（`DecoratorNames.ONCE = 'Once'`）
## 源码参考位置
- 动态：`compiler/src/validate_ui_syntax.ts:1137`（`validateSingleDecorator`，@Once 目标校验） 中的 `@Once` 装饰器校验逻辑
- 静态：`arkui-plugins/collectors/ui-collectors/validators/rules/check-once-decorator.ts:61`

## 适用对象
- struct/class 的成员属性
- struct/class 的成员方法（`@Once` 不可用于方法）

## 报错信息
- 动态：
  - `'@Once' can only decorate member property.`（错误码 10905346）
- 静态：
  - `'@Once' can only decorate member property.`
  - `When a variable decorated with '@Once', it must also be decorated with '@Param'.`

## 错误码
- 10905346：@Once 用于非成员属性

## 核心校验规则
1. `@Once` 只能装饰成员属性（不可用于方法、函数、接口等）
2. 使用 `@Once` 时必须同时使用 `@Param`（若未使用 `@Param`，提供添加 `@Param` 的自动修复）
3. `@Once` 只能在 struct 中使用（由 `check-validate-decorator-target` 校验）
4. `@Once` 用于 `@ComponentV2` 装饰的 struct（由 `check-old-new-decorator-mix-use` 校验）
5. 校验等级为 ERROR（阻断编译）

## 校验实现细节
### 静态工具链实现
源码位于 `check-once-decorator.ts`，采用按节点类型分发的策略：

1. **节点类型分发**：`_checkOnceDecorator` 根据 `arkts.nodeType(node)` 从 `checkByType` Map 中选择处理函数：
   - `AST_NODE_TYPE_METHOD_DEFINITION` -> `checkOnceInMethod`
   - `AST_NODE_TYPE_CLASS_PROPERTY` -> `checkOnceInProperty`

2. **方法节点校验**（`checkOnceInMethod`）：
   - 区分 struct 方法和普通 class 方法，分别走 `checkOnceInStructMethod` / `checkOnceInClassMethod`
   - 从 `metadata.ignoredAnnotationInfo?.hasOnce` 判断是否存在 `@Once`
   - 从 `metadata.ignoredAnnotations?.[DecoratorNames.ONCE]` 获取装饰器节点
   - 若 `@Once` 存在但 `@Param` 不存在：调用 `hasRawParamAnnotation(node)` 二次确认原始 AST 是否含 `@Param`（防止元数据未同步），若无则报告"必须同时使用 @Param"并提供添加 `@Param` 的修复建议
   - 若 `@Once` 和 `@Param` 同时存在于方法上：报告"`@Once` 只能装饰成员属性"并提供移除 `@Once` 装饰器的修复建议

3. **属性节点校验**（`checkOnceInProperty`）：
   - 区分 struct 属性和普通 class 属性
   - struct 属性从 `metadata.annotationInfo?.hasOnce` 和 `metadata.annotations?.[DecoratorNames.ONCE]` 获取
   - class 属性从 `metadata.ignoredAnnotationInfo?.hasOnce` 和 `metadata.ignoredAnnotations?.[DecoratorNames.ONCE]` 获取
   - 逻辑与方法节点类似：若无 `@Param` 则报告并建议添加 `@Param`

4. **`hasRawParamAnnotation` 辅助函数**：直接检查 AST 节点的 `annotations` 字段（属性节点）或 `node.function.annotations`（方法节点）中是否存在 `@Param`，作为元数据检查的兜底

### 动态工具链实现
`compiler/src/validate_ui_syntax.ts:1137`（`validateSingleDecorator`，@Once 目标校验） 校验 `@Once` 只能用于成员属性，错误码 10905346。

## 适用场景
- `@Once` 是 V2 状态管理装饰器，用于标记 `@Param` 参数仅初始化一次，后续父组件更新不再同步
- 必须与 `@Param` 配合：`@Param` 负责接收父组件传值，`@Once` 负责切断后续同步
- 仅适用于 `@ComponentV2` 装饰的 struct 或普通 class 成员属性
- 不可与 V1 装饰器（`@Prop`、`@Link` 等）混用

## 自动修复建议
### 修复1：添加 `@Param` 装饰器
当 `@Once` 存在但 `@Param` 缺失时，修复建议在 `@Once` 装饰器末尾位置插入 ` @Param`：
```typescript
// 修复前
@Once count: number = 0
// 修复后
@Param @Once count: number = 0
```
修复动作由 `createSuggestion(' @Param', endPosition, endPosition, ADD_PARAM_ANNOTATION)` 生成。

### 修复2：移除 `@Once` 装饰器
当 `@Once` 用于方法时，修复建议删除整个 `@Once` 装饰器：
```typescript
// 修复前
@Once
onMethod(): void { }
// 修复后
onMethod(): void { }
```
修复动作由 `createSuggestion('', ...getPositionRangeFromAnnotation(onceAnnotation), REMOVE_ANNOTATION)` 生成，范围通过 `getPositionRangeFromAnnotation` 计算装饰器完整覆盖区域。

## 示例代码
### 反例
```typescript
@ComponentV2
struct MyComp {
  @Once count: number = 0          // @Once 未配合 @Param

  @Once                           // @Once 用于方法
  onMethod(): void { }
}
```

### 正例
```typescript
@ComponentV2
struct MyComp {
  @Param @Once count: number = 0  // @Once 配合 @Param
}
```
